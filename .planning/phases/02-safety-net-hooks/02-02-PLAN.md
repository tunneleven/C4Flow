<objective>
Create the hook test suite that validates all three hooks work correctly via mock stdin and file fixtures, then verify hooks behave as expected via human testing.

Purpose: Hooks are the safety net — if they fail silently, agents can bypass quality gates. The test suite provides regression testing without needing a live Claude Code session, by piping mock JSON stdin to the hook scripts and checking exit codes and stdout/stderr output.

Output: Six test scripts in .claude/tests/ plus a human verification checkpoint confirming hook behavior in a real project context.
</objective>

Create `.claude/tests/` directory.

All test scripts follow this pattern: set up a temp directory with fixtures, run the hook script with mock stdin, check exit code and output, clean up. Use a simple PASS/FAIL counter. Each test script is self-contained and exits non-zero if any test fails.

1. **test-hook-bd-close.sh** — Tests for HOOK-01 (PreToolUse bd-close-gate):
   - Setup: create temp dir, create `.bd` file (beads project marker), create `quality-gate-status.json` fixture with `overall_pass: false`
   - Test 1: "Denies bd close when gates not passed" — pipe `{"tool_name":"Bash","tool_input":{"command":"bd close task-001 --reason done"}}` to the hook script (run from temp dir). Assert exit 0 AND stdout contains `"permissionDecision"` AND stdout contains `"deny"`.
   - Test 2: "Allows bd close when gates passed" — update fixture to `overall_pass: true`. Pipe same input. Assert exit 0 AND stdout is empty (no JSON output means allow).
   - Test 3: "Allows bd close --force regardless" — reset fixture to `overall_pass: false`. Pipe input with `"command":"bd close task-001 --force --reason skip"`. Assert exit 0 AND stdout is empty.
   - Test 4: "Allows non-bd-close commands" — pipe `{"tool_name":"Bash","tool_input":{"command":"ls -la"}}`. Assert exit 0 AND stdout is empty.
   - Test 5: "Denies when gate status file missing" — remove `quality-gate-status.json`. Pipe bd close input. Assert stdout contains `"deny"`.
   - Test 6: "Skips on non-beads project" — remove `.bd` file. Pipe bd close input. Assert exit 0 AND stdout is empty.
   - Cleanup: remove temp dir.
   - IMPORTANT: Run each test by `cd`-ing to the temp dir and invoking the hook script via absolute path. Set `CLAUDE_PROJECT_DIR` env var to the temp dir for path resolution. Override `PATH` or use function mock if `jq` behavior needs to be controlled.

2. **test-hook-stop.sh** — Tests for HOOK-02 (Stop check-open-gates):
   - This hook calls `bd gate list --json` which requires a real beads DB. Since we cannot create a real beads environment in tests, mock the `bd` command:
   - Setup: create temp dir, create `.bd` file, create a mock `bd` script in temp dir that outputs canned JSON when called with `gate list --json`. Add mock bd to PATH.
   - Test 1: "Blocks when c4flow gates open" — mock bd returns `[{"id":"bd-001","title":"Quality gate","labels":["c4flow-quality-gate"]}]`. Run hook from temp dir. Assert stdout contains `"block"` and `"bd-001"`.
   - Test 2: "Allows when no c4flow gates" — mock bd returns `[{"id":"bd-002","title":"Other gate","labels":["unrelated"]}]`. Run hook. Assert exit 0 AND stdout is empty.
   - Test 3: "Allows when gate list empty" — mock bd returns `[]`. Assert exit 0.
   - Test 4: "Handles null labels gracefully" — mock bd returns `[{"id":"bd-003","title":"No labels"}]` (no labels field). Assert exit 0 AND no jq errors on stderr.
   - Test 5: "Skips on non-beads project" — remove `.bd` file. Assert exit 0.
   - Cleanup: remove temp dir, restore PATH.

3. **test-hook-taskcompleted.sh** — Tests for HOOK-03 (TaskCompleted task-complete-gate):
   - Setup: create temp dir, create `.bd` file, create `quality-gate-status.json` fixture with `overall_pass: false`. Create mock `bd` in PATH (needed for the missing-file fallback check).
   - Test 1: "Blocks when gates not passed" — pipe `{"task_subject":"Implement auth"}` to hook. Assert exit 2 AND stderr contains "quality gates not passed".
   - Test 2: "Allows when gates passed" — update fixture to `overall_pass: true`. Assert exit 0.
   - Test 3: "Blocks when file missing but gates exist" — remove gate status file, mock bd returns gates. Assert exit 2.
   - Test 4: "Allows when file missing and no gates" — mock bd returns `[]`. Assert exit 0.
   - Test 5: "Skips on non-beads project" — remove `.bd`. Assert exit 0.
   - Cleanup.

4. **test-hook-files.sh** — Tests for INFR-02:
   - Test that `.claude/hooks/bd-close-gate.sh` exists and is executable (`test -x`)
   - Test that `.claude/hooks/check-open-gates.sh` exists and is executable
   - Test that `.claude/hooks/task-complete-gate.sh` exists and is executable
   - Test that each file starts with `#!/usr/bin/env bash`

5. **test-hook-settings.sh** — Tests for INFR-03:
   - Test that `.claude/settings.json` is valid JSON (`jq . .claude/settings.json`)

   - Test that `hooks.Stop` exists

   - Test that `enabledPlugins` still exists (merge did not clobber)
   - Test that all hook command paths end with the correct script names

6. **run-hooks-tests.sh** — Suite runner:
   - Shebang, set -euo pipefail
   - Run each test script in order, track pass/fail count
   - Print summary at end
   - Exit non-zero if any test script failed
