<objective>
Create the three Claude Code hook scripts and register them in .claude/settings.json to intercept agent-initiated quality gate bypasses.

Purpose: These hooks form the safety net layer that prevents Claude agents from closing beads tasks, ending sessions, or marking tasks complete when quality gates are still open. They read state established in Phase 1 (quality-gate-status.json) and the beads gate list — they never write state.

Output: Four files — settings.json (updated with hooks config), bd-close-gate.sh, check-open-gates.sh, task-complete-gate.sh — all executable and project-scoped.
</objective>

   - Stop: command "$CLAUDE_PROJECT_DIR/.claude/hooks/check-open-gates.sh", timeout 30

   - If file not found: output JSON deny with reason "No quality gate status found. Run /c4flow:review before closing this task." and exit 0
   - If `overall_pass` is `true`: exit 0 (allow close)
   - Otherwise: build denial message showing gate_id, codex_review status (PASS/FAIL/NOT RUN), bd_preflight status (PASS/FAIL/NOT RUN), and remediation steps. Output as `hookSpecificOutput` JSON with `permissionDecision: deny`. Exit 0.
   - Use the exact script skeleton from 02-RESEARCH.md Pattern 1 but apply the path resolution fix from Pitfall 4 (use CLAUDE_PROJECT_DIR with git-toplevel fallback instead of bare $(pwd)).

4. `chmod +x .claude/hooks/bd-close-gate.sh` (Pitfall 6 — scripts must be executable).
