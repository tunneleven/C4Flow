  <action>
Replace the placeholder `skills/code/SKILL.md` with a full CODE-phase skill document.

Write the skill as a step-by-step orchestration guide that does all of the following with explicit paths and commands:

1. Frontmatter:
   - `name: c4flow:code`
   - `description: Execute code implementation via Superpowers subagent-driven workflow`

2. Overview section:
   - State that CODE is Phase 3: Implementation.
   - State that TDD behavior is merged into CODE through downstream Superpowers execution.
   - State that this skill is a coordinator, not the per-task implementer.

3. Prerequisite checks section:
   - Read `docs/c4flow/.state.json`.
   - Confirm `currentState` is `CODE` when invoked from the orchestrator, or explain that direct invocation is allowed for recovery/manual resume.
   - Check for implementation input from beads or `docs/specs/<feature>/tasks.md`.
   - Check for an execution plan or planning artifact before dispatching work.
   - If prerequisites are missing, print exact recovery commands such as `/c4flow:beads`, `$gsd-plan-phase`, or the relevant file path to create.

4. Superpowers integration section:
   - Explicitly instruct the operator to load `using-superpowers` first.
   - Explicitly route implementation through `subagent-driven-development`.
   - Mention `using-git-worktrees` as a required setup step before execution begins.
   - Mention that downstream implementation/review agents follow the review loops defined by `subagent-driven-development`.

5. Execution flow section:
   - Describe the route as: validate state → confirm tasks/plan → enter Superpowers workflow → run subagent-driven implementation → verify all assigned tasks are closed → advance to TEST.
   - Keep CODE → TEST contingent on task completion, not merely on invoking the skill.
   - Include exact state-file fields to update when CODE is finished: `currentState = "TEST"`, append `"CODE"` to `completedStates`, reset `failedAttempts` to `0`, clear `lastError`.

6. Fallback and failure handling:
   - If Superpowers skills are unavailable, instruct manual execution from the existing plan rather than silently proceeding.
   - If execution is blocked, direct the user back to the prior missing artifact or planning step.

The finished file must be at least 120 lines and must contain the literal strings `using-superpowers`, `subagent-driven-development`, `docs/c4flow/.state.json`, and `currentState`.
