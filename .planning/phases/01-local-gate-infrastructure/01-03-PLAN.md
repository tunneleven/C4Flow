```yaml
name: c4flow:verify
description: Quality gate aggregation — runs bd preflight, combines with Codex review results, declares Ready for PR status
---
```

**Skill body must implement these steps IN ORDER:**

**Step 1: Tool availability detection (GATE-04 partial)**
- Check `command -v bd`
- If bd missing: warn user with install URL (https://github.com/steveyegge/beads). Print manual preflight checklist as fallback (check for uncommitted changes, check for TODO markers, check branch is up to date). Exit gracefully
- If bd available: proceed

**Step 2: Check existing gate status for expiry**
- Read `quality-gate-status.json` if it exists
- If file exists, check `expires_at` field. If expired: warn "Gate status expired. Run `/c4flow:review` first to refresh."
- If file does not exist: warn "No gate status found. Run `/c4flow:review` first." Exit with status summary showing all checks as "not run"

**Step 3: Run bd preflight (GATE-02)**
- Execute: `bd preflight --check --json 2>&1`
- Capture full output
- Parse JSON: extract `pass` boolean and `issues` array
- If JSON parse fails: treat as preflight failure, set `pass: false`, `issues: [{"check": "parse_error", "message": "Failed to parse bd preflight output"}]`

**Step 4: Write preflight results to quality-gate-status.json**
- Use atomic write pattern (same as review skill): read existing -> merge -> write temp -> mv
- Update `checks.bd_preflight` section with:
  - `pass`: boolean from preflight result
  - `ran_at`: current ISO 8601 timestamp
  - `issues`: array from preflight output
- Preserve existing `checks.codex_review` results (do NOT overwrite)
- Update `generated_at` and `expires_at` (refresh expiry on new data)
- Recompute `overall_pass`: true ONLY if `codex_review.pass === true AND bd_preflight.pass === true`. If either is null or false, `overall_pass` must be false

**Step 5: Gate resolution check**
- Read `gate_id` from `quality-gate-status.json`
- **Fallback if gate_id is null:** attempt label-based lookup: `bd gate list --json | jq -r '.[] | select(.labels[] | contains("c4flow-quality-gate")) | .id' | head -1`. If found, update gate_id in the status file
- If `overall_pass` is now true AND `gate_id` is non-null:
  - Resolve the gate: `bd gate resolve "$GATE_ID" --reason "All checks passed. Codex: 0 critical, 0 high. Preflight: passed. Verified at: $TIMESTAMP"`
  - Report: "All quality gates resolved. Ready for PR."
- If `overall_pass` is false:
  - Do NOT resolve the gate
  - Report which checks failed and why

**Step 6: Aggregated summary output**
- Print a clear status report:
  ```
  === C4Flow Quality Gate Status ===

  Codex Review:  [PASS/FAIL/NOT RUN]  (ran: <timestamp> | not yet run)
    Critical: N  High: N  Medium: N  Low: N
  bd Preflight:  [PASS/FAIL/NOT RUN]  (ran: <timestamp> | not yet run)
    Issues: N

  Ready for PR: YES / NO

  Gate ID: <gate_id>
  Expires: <expires_at>
  ```
- If Ready: YES — print:
  - "Run `/c4flow:pr` to create the pull request."
  - **IMPORTANT (INFR-04 audit trail):** Also print a reminder checklist item:
    ```
    REMINDER: When closing this task with `bd close`, you MUST include --reason:
      bd close <bead_id> --reason "Completed: <brief summary of what was done>"
    This is required for audit trail compliance.
    ```
- If Ready: NO — print which checks need attention:
  - If codex_review not run: "Run `/c4flow:review` first"
  - If codex_review failed: "Fix CRITICAL/HIGH findings, then re-run `/c4flow:review`"
  - If bd_preflight failed: "Fix preflight issues, then re-run `/c4flow:verify`"
  </action>
