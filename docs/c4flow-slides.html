Deep Dive

Tại sao Beads, không phải Markdown?

🔴 Agent Amnesia

Agent làm Task 1 được 50% → Context window đầy → Compact → Đọc lại tasks.md → Thấy "- [ ] Task 1" (chưa check!) → LÀM LẠI TỪ ĐẦU! 😱

🔴 Implicit Dependencies

"depends on" nghĩa là block hay chỉ liên quan?

Suy luận = tốn tokens + có thể sai

🔴 Chỉ 2 trạng thái

[ ] TODO vs [x] DONE

Không có: "đang làm 50%", "đang đợi review", "bị block"

🔴 Không scale

File 500 dòng = 2000 tokens mỗi lần đọc

Agent đọc 3 lần = 6000 tokens chỉ để biết task nào cần làm

  • Code phase nhanh — design đã quyết

Phase 4–5

Beads Breakdown → TDD Code Loop

📦 Task Breakdown

/c4flow:beads # Đọc spec → tạo epic + dependency graph # Chọn: compact | balanced | atomic # Agent ước lượng → bạn approve

🔄 Code Loop

bd ready → bd update --status in_progress → TDD (RED→GREEN→REFACTOR) → Test → Review → Verify → bd close → Next task

🧪 TDD bắt buộc

RED — Test fail trước GREEN — Code vừa đủ pass REFACTOR — Clean up

🔄 Recovery sau Compact

$ bd ready --json # Không có task ready $ bd list --status in_progress # Tìm task đang làm dở $ bd show bd-1 # Đọc notes → biết đang ở đâu → TIẾP TỤC ĐÚNG CHỖ!

Phải --force để bypass → logged!

Tại sao quan trọng?

  • ⚡ Tốc độ ≠ bỏ qua chất lượng
  • 🛡️ Quality gates đảm bảo code merge-ready
  • 👨‍⚖️ Ban giám khảo thấy: code có test, có review
  • 🔒 Ngay cả khi bạn muốn skip — gates ngăn chặn
